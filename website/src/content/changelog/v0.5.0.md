The first milestone that a developer can actually use. Given a Docker image, AppBahn
creates a Deployment, a Service, and an Ingress, requests a TLS certificate, and
returns a working URL. No Kubernetes manifests written by hand.

### What shipped

- Resource creation flow (console + CLI) for Docker image sources.
- Operator reconciles Deployment/Service/Ingress from a single Resource CR.
- cert-manager integration for auto-issued TLS.
- Auto-generated domain scheme: `{resource}.{env}.{cluster-domain}`.

### Why this matters

Everything before this release was scaffolding. This is the first point at which
AppBahn stops being "a platform for platform engineers to evaluate" and starts being
"a platform developers can push to." The pipeline is still image-only — Git-to-deploy
is a later release — but the deploy path itself is complete.
