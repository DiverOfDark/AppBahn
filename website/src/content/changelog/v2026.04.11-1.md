The first point at which AppBahn stops being "a platform for platform engineers
to evaluate" and starts being "a platform developers can push to." Given a Docker
image, AppBahn creates a Deployment, a Service, and an Ingress, requests a TLS
certificate, and returns a working URL. No Kubernetes manifests written by hand.

This is also the first tagged public build — everything that landed before this
is reflected in the earlier entries below.

### What shipped

- Resource creation flow (console + CLI) for Docker image sources.
- Operator reconciles Deployment/Service/Ingress from a single Resource CR.
- cert-manager integration for auto-issued TLS.
- Auto-generated domain scheme: `{resource}.{env}.{cluster-domain}`.
- Automated GitHub releases: every merge to `master` produces a dated tag
  (`vYYYY.MM.DD-N`) with signed container images and a Helm chart bump.

### Why this matters

Everything before this release was scaffolding. This is the first point at which
the deploy path itself is complete — and the first point at which a user can
install a tagged, signed build instead of cloning from `master`.
